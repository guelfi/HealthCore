# Work Session Protocol

## Start

1. Read `STATUS.md`, `BACKLOG.md`, `RISKS.md`, and the previous session.
2. Confirm the branch and workspace-only scope.
3. Record Git status, baseline commit, objective, and planned verification.
4. Create a session file from `sessions/TEMPLATE.md`.
5. Mark only one backlog item as `in_progress` where practical.

## During Work

1. Work in small, reviewable batches.
2. Update the session after meaningful discoveries or validations.
3. Record decisions and risks in their authoritative files.
4. Never mark work done without verification evidence.
5. Stop at mandatory approval gates.

## Close

1. Run all checks affected by the session changes in containers.
2. Record commands, results, changed paths, and unresolved issues.
3. Update backlog, status, risks, removals, and verification evidence.
4. State the exact first action for the next session.
5. Leave a clean worktree or explicitly document every uncommitted path.
